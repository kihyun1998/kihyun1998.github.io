# **Section 23.** 객체 지향 ( Object-Oriented )
---

## 객체 지향

객체들을 조립하듯 소프트웨어 개발.

### 객체 지향 특징

- 구조적 기법의 문제점에 대한 해결책
- 재사용 및 확장 용이, 유지보수 쉽다.
- 구조를 단계적, 계층적으로 표현 
- 멀티미디어 데이터 and 병렬 처리 지원
- 사용자와 개발자 쉽게 이해


## 객체 ( Object )

데이터와 함수를 묶어 놓은 모듈
>`메시지` : 객체에게 지시하는 명령

### 객체의 특성

- 독립적으로 식별 가능한 이름
- 일반적으로 시간에 따라 변함
- 다른 객체와 상호 연관성에 의한 관계 형성
- 메시지의 집합 > 행위
- 일정한 기억장소 보유


## 클래스 ( Class )

하나 이상의 유사한 객체를 묶어서 하나의 공통된 특성을 표현한 것

### 클래스 특성

- 객체들이 갖는 속성과 연산을 정의하는 틀
- 추상화하는 단위
- 클래스에 속한 객체를 인스턴스라고 함.
- Super Class > 부모 클래스 , Sub Class > 자식 클래스


## 캡슐화 ( Encapsulation )

데이터와 함수를 하나로 묶는 것

### 캡슐화 특징

- 캡슐화된 객체는 정보은닉되어 접근 제한적임
- 재사용이 용이
- 인터페이스 단순해지고 객체 간의 결합도 낮아짐


## 상속 ( Inheritance )

부모 클래스의 모든 속성과 연산을 하위 클래스로 물려받는 것

### 상속 특징

- 하위 클래스에서 상속받은 속성, 연산을 제외하고도 새로 추가할 수 있음
- 소프트웨어 재상요을 높인다
>`다중상속` : 한 개의 클래스가 두 개 이상의 부모 클래스로부터 상속 받는 것


## 다형성 ( Polymorphism )

하나의 메시지에 대해 여러 가지 형태의 응답이 있다는 것을 의미

### 다형성 특징

- 동일한 메소드 명을 사용하여 같은 의미 응답을 함
> `오버 로딩` : 메소드의 이름은 같지만 인수를 받는 자료형과 개수를 달리하여 여러 기능 정의
> `오버 라이딩` : 상위 클래스에서 정의한 메소드와 이름은 같지만 메소드 안의 실행 코드를 달리하여 자식 클래스에서 재정의하여 사용


## 연관성 ( Relationship )

두 개 이상의 객체들이 상호 참조하는 관계

### 연관성 종류

- 연관화 ( Association )
    - `is member of`
    - 2개 이상의 객체가 상호 관련되어 있음
- 분류화 ( Classfication )
    - `is instance of`
    - 동일한 형의 특성을 갖는 객체를 모음
- 집단화 ( Aggregation )
    - `in part of`
    - 관련 있는 객체를 묶어 하나의 상위 객체 구성
- 일반화 ( Generalization )
    - `is a`
    - 공통적인 성질들로 추상화한 상위 객체 구성
- 특수화 / 상세화 ( Specialization )
    - `is a`
    - 상위 객체를 구체화하여 하위 객체를 구성
