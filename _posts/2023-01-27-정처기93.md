# **Section 93.** 트랜잭션 분석 / CRUD 분석
---


## 트랜잭션(Transaction)이란

데이터베이스의 상태를 변환시키는 하나의 논리적 기능을 수행하기 위한 **작업의 단위**, 수행되어야 할 연산

- 데이터베이스 시스템에서 병행 제어 및 회복 작업 시 처리되는 작업의 논리적 단위로 사용됨
- 사용자가 시스템에 대한 서비스 요구 시 시스템이 응답하기 위한 상태변환 과정의 작업 단위로 사용됨.


### 트랜잭션 상태

|이름|설명|
|---|------|
|활동(Active)|트랜잭션이 실행 중인 상태|
|실패(Failed)|트랜잭션 실행에 오류가 발생하여 중단된 상태|
|철회(Aborted)|트랜잭션이 비정상적으로 종료되어 Rollback 연산을 수행한 상태|
|부분 완료(Partially Committed)|트랜잭션을 모두 성공적으로 실행한 후 Commit 연산이 실행되기 직전인 상태|
|완료(Committed)|트랜잭션을 모두 성공적으로 실행한 후 Commit 연산을 실행한 후 상태|


### 트랜잭션의 특성

무결성을 보장하기 위해 트랜잭션이 가져야 할 특성들

- **원자성** (Atomicity) :
    - 트랜잭션 연산은 모두 반영되도록 완료(Commit)되던지 전혀 반영되지 않도록 복구(Rollback)되어야 함.
    - 모든 명령은 반드시 완벽히 수행되어야 함. 오류 발생하면 트랜잭션 전부가 취소돼야 함
    - 정리 : 다 되거나 아님 말거나

- **일관성** (Consistency) :
    - 실행을 성공적으로 완료하면 일관성 있는 데이터베이스 상태로 변환함.
    - 시스템이 갖고 있는 **고정 요소**는 트랜잭션 수행 전과 트랜잭션 수행 완료 후의 상태가 같아야 함.
    - 정리 : 트랜잭션 전과 후 같아야 함(고정 요소)

- **독립성** (Isolation, 격리성, 순차성) :
    - 둘 이상 트랜잭션 병행 실행 시 하나의 트랜잭션 실행 중 끼어들 수 없음.
    - 수행중인 트랜잭션이 완전히 완료될 때까지 다른 트랜잭션에서 결과 참조 못함
    - 정리 : 순차적

- **영속성** (Durability, 지속성) :
    - 성공적으로 완료된 트랜잭션의 결과는 시스템이 고장나도 영구적 반영 되어야함.

## CRUD 분석

생성(Create), 읽기(Read), 갱신(Update), 삭제(Delete)의 약자로 CRUD 연산에 대해 CRUD 매트릭스를 작성하여 분석하는 것
**목적** : 테이블에 발생하는 트랜잭션을 분석하여 물리적인 데이터베이스 설계 시 구조를 최적화하기 위함 

- 테이블에 발생되는 트랜잭션의 주기별 발생 횟수 파악, 저장되는 데이터 양 유추
- 많은 트랜잭션이 몰리는 테이블 파악 가능
- 외부 프로세스 트랜잭션의 부하가 집중되는 데이터베이스 채널 파악하고 분산하여 연결 지연 또는 타임아웃 오류 방지


### CRUD 매트릭스

2차원 형태의 표로 행에는 프로세스, 열에는 테이블이 있는 상관 분석표

- 트랜잭션이 테이블에 수행하는 작업을 검증함.
- C > D > U > R의 우선순위를 적용하여 한 가지 or 모두 적음
- CRUD가 적히지 않은 행,열을 찾거나 C나 R이 없는 열을 확인하여 불필요하거나 누락된 테이블 또는 프로세스 탐색
> CRUD중 아무것도 없으면 사용하지 않는 테이블이니 삭제를 검토, C가 없다면 입력된게 없지만 사용, R이 없으면 입력은 하지만 사용하지 않음

## 트랜잭션 분석

CRUD 매트릭스를 기반으로 테이블에 발생하는 트랜잭션 양을 분석하여 테이블에 저장되는 **데이터의 양을 유추**하고 이를 근거로 **DB 크기 선정** 및 **DB 구조 최적화**

- 트랜잭션 분석은 업무 개발 담당자가 수행
- 프로세스가 과도하게 접근하는 테이블을 확인하여 여러 디스크를 배치하여 분산하여 성능 향상을 가져올 수 있음


### 트랜잭션 분석서

**단위 프로세스** : 업무를 발생시키는 가장 작은 단위의 프로세스
**CRUD 연산** : CRUD 연산
**테이블명, 칼럼명** : 프로세스가 접근하는 테이블명, 필요한 경우 칼럼명도 적음 ex) '테이블명.칼럼명'
**테이블 참조 횟수** : 프로세스가 테이블 참조한 횟수
**트랜잭션 수** : 주기별로 수행되는 트랜잭션 횟수
**발생 주기** : 트랜잭션 횟수를 측정하기 위한 발생 주기
