# **Section 05.** 현행 시스템 파악
---

## 개발 기술 환경이란?

개발하고자 하는 소프트웨어와 관련된 `운영체제`, `DBMS`, `미들웨어` 등을 선정할 때 고려해야 할 사항 기술
> `미들웨어` : 운영체제와 실행되는 응용 프로그램 사이에서 운영체제가 제공하는 서비스 외의 추가적 서비스 제공 소프트웨어



## 운영체제
---

### 운영체제 개념

- 컴퓨터 시스템의 자원들을 효율적으로 관리
- 사용자가 컴퓨터를 효율적으로 사용할 수 있도록 환경 제공
- 소프트웨어

### 운영체제 특징

- 사용자와 하드웨어 간의 인터페이스로서 시스템 소프트웨어의 일종
- 환경 제공
- 컴퓨터 운영체제 : Windows, UNIX, Linux, Mac OS
- 모바일 운영체제 : iOS, Android

### 운영체제 관련 요구사항 식별 시 고려사항

- **가용성** : 
    - 장시간 운영에 따른 고유 장애 발생 가능성
    - 메모리 누수로 인한 성능 저하 및 재가동
    - 보안 패치로 인한 재가동
    - 결함 패치로 인한 재가동
> `가용성` : 실행하고 싶을 때 실행되는지에 대한 능력
- **성능** :
    - 대규모 동시 사용자에 대한 처리
    - 대규모 및 대용량 파일 처리
    - 지원 메모리 크기 ( 32bit, 64bit )
- **기술 지원** :
    - 제작 업체의 기술 지원
    - 정보 공유
    - 오픈소스 여부
- **주변기기**
    - 설치 가능한 하드웨어
    - 주변기기 지원 여부
- **구축비용**
    - 지원 가능한 하드웨어 비용
    - 라이선스 정책 및 비용
    - 유지관리비
    - 총 소유 비용
    > `총 소유 비용(TCO)` : Total Cost of Ownership으로 어떤 자산을 획득하려할 때 지정된 기간동안 발생할 수 있는 모든 직간접 비용



## 데이터베이스 관리 시스템( DBMS )
---

### DBMS 개념

- 사용자와 DB 사이에서 사용자 요구에 따라 정보 생성
- DB 관리 소프트웨어

### DBMS 특징

- DBMS는 DB의 구성, 접근 방법, 유지관리에 대한 모든 책임을 짐.
- DBMS 종류에는 Oracle, IBM DB2, Microsoft SQL Server, MySQL, SQLite, MongoDB, Redis 등

### DBMS 관련 요구사항 식별 시 고려사항

- **가용성** : 
    - 장시간 운영에 따른 고유 장애 발생 가능성
    - DBMS 결함 패치 시 재가동
    - 백업, 복구의 편의성
    - DBMS 이중화 및 복제 지원
- **성능** :
    - 대규모 데이터 처리 성능
    - 분할 테이블 지원 여부
    - 대용량 트랜잭션 처리 성능
    - 튜닝 옵션의 다양한 지원
    - 최소화된 설정과 비용 기반 질의 최적화 지원
    > `비용 기반 질의 최적화` : 사용자가 질의에 대한 최적의 실행 방법을 결정하기 위한 것 필요한 소요 시간과 자원 사용량이 최소인 방법을 선택
- **기술 지원** :
    - 제작 업체의 기술 지원
    - 정보 공유
    - 오픈소스 여부
- **상호 호환성**
    - 설치 가능한 운영체제 종류
    - JDBC, ODBC와의 호환 여부
    > `JDBC` : Java-DB-Connectivity로 자바에서 DB에 접근하여 데이터를 CRUD할 수 있도록 연결해주는 인터페이스
    > `ODBC` : Open-DB-Connectivity로 응용프로그램에서 CRUD할 수 있도록 인터페이스
- **구축 비용**
    - 라이선스 정책 및 비용
    - 유지관리비
    - 총 소유 비용



## 웹 어플리케이션 서버( WAS : Web Application Server )
---

### WAS 개념

- 사용자의 요구에 따라 변하는 동적인 콘텐츠를 처리하기 위해 사용되는 미들웨어

### WAS 특징

- 데이터 접근, 세션 관리, 트랜잭션 관리 등을 위한 라이브러리 제공
- 주로 DB 서버와 연동하여 사용
- 종류에는 Tomcat, GlassFish, JBoss, Jetty, JEUS, Resin, WebLogic, WebSphere 등이 있다.

### WAS 관련 요구사항 식별 시 고려사항

- **가용성** : 
    - 장시간 운영에 따른 고유 장애 발생 가능성
    - WAS 결함 패치 시 재가동
    - 안정적인 트랜잭션 처리
    - WAS 이중화 지원
- **성능** :
    - 대규모 트랜잭션 처리 성능
    - 다양한 설정 옵션 지원
    - 가비지 컬렉션의 다양한 옵션
    > `가비지 컬렉션` : Garbage Collection으로 실제 사용되지 않으면서 가용 공간 리스트에 반환되지 않는 메모리 공간
- **기술지원**
    - 제작 업체의 기술 지원
    - 정보 공유
    - 오픈소스 여부
- **구축 비용**
    - 라이선스 정책 및 비용
    - 유지관리비
    - 총 소유 비용



## 오픈소스
---

### 오픈소스 개념

오픈소스는 누구나 제한 없이 사용할 수 있도록 소스코드를 공개한 것

### 오픈소스 고려사항

- 라이선스의 종류
- 사용자 수
- 기술의 지속 가능성
