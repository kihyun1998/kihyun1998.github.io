# **Section 191.** 암호 알고리즘
---


## 암호 알고리즘 이란?

패스워드, 주민번호 등 중요 정보를 보호하기 위해 평문을 암호화된 문장으로 만드는 절차 or 방법

- 해시를 사용하는 단방향 암호 방식과 개인키 및 공개키를 사용하는 양방향 암호화 방식이 있음

- `단방향` : HASH
- `양방향` :
    - **개인키** : Stream 방식, Block 방식
    - **공개키**


## 개인키 암호화 기법

동일한 키로 데이터를 암호화하고 복호화하는 기법

- 암호화 대상이 n개일 때 사용되는 키의 수는 n(n-1)/2이다.
- **블록 암호화 방식** : 한 번에 하나의 데이터 블록을 암호화
    - ex) DES, SEED, AES, ARIA, IDEA
- **스트림 암호화 방식** : 평문과 동일한 길이의 스트림 생성하여 비트/바이트/워드 단위로 암호화
    - ex) LFSR, RC4
- **장점** : 
    - 속도 빠름
    - 알고리즘 단숨
    - 파일 크기 작음
- **단점** :
    - 사용자가 증가하면 관리해야할 키의 수가 상대적으로 많아짐


## 공개키 암호화 기법

암호화할 때 사용하는 공개키는 사용자들에게 공개하고 복호화할 때 사용하는 개인키는 관리자가 관리함

- 암호화 대상이 n개면 사용되는 키의 수는 2n이다.
- 비대칭 암호 기법이라고도 하며 대표적으로 RSA기법이 있음
- 인증, 전자서명도 가능
- **장점** :
    - 키 분배 용이 
    - 관리해야할 키 수가 적음
- **단점** :
    - 암/복호화 속도가 느림
    - 알고리즘 복잡
    - 파일 크기 큼


### 공개키 기반 구조 (PKI; Public Key Infrastructure)

공개키 암호 시스템을 안전하게 사용하고 관리하기 위한 정보 보호 표준 방식

- X.509 방식 : 인증 기관에서 발생하는 인증서를 기반으로 상호 인증 제공
- 비X.509 방식 : 국가별, 지역별로 맞게 보완 및 개발



## 양방향 알고리즘 정리

- `SEED` :
    - KISA에서 개발한 블록 암호화 알고리즘
    - **블록 크기** : 128
    - **키 길이** : 128, 256
- `ARIA` (Academy, Research Institute, Agency) :
    - 국정원 개발 블록 암호화 알고리즘
    - 학계, 연구기관, 정부의 영문 앞글자 땀
    - **블록 크기** : 128비트
    - **키 길이** : 128, 292, 256
- `DES` (Data Encryption Standard) : 
    - 미국 NBS에서 발표한 개인키 암호화 알고리즘
    - 3DES : DES 3번 적용한거
    - **블록 크기** : 64비트
    - **키 길이** : 56비트
- `AES` (Advanced Encryption Standard) :
    - 미국 NIST에서 발표 개인키 암호화 알고리즘
    - DES에 한계 느끼고 발표
    - **블록 크기** : 128비트
    - **키 길이** : 128, 192, 256
- `RSA` (Rivest Shamir Adlemna) :
    - MIT의 라이베스트, 샤미르, 애들먼의 제안된 공개키 암호화 알고리즘
    - 큰 숫자를 소인수분해해서 사용
    - 공개키와 비밀키 사용

## 해시

입력 데이터를 고정된 길이의 값으로 변환하는 것

- 해시 알고리즘을 해시 함수라고 함, 변환된 값은 해시 값이라 함
- 무결성 검증으로 사용하거나 정보보호의 다양한 분야에서 사용함
- 복호화가 거의 불가능한 일방향 함수다


### 해시 함수의 종류

- `SHA 시리즈` :
    - 미국 NSA가 처음 설계, NIST에 의해 발표
    - SHA-0부터 SHA-1,2,224,256,384,512가 발표
- `MD5` :
    - 1991 / R.Rivest가 MD4를 대체하기 위해 발표
    - **블록 크기** : 512비트
    - **키 길이** : 128비트
- `N-NASH` :
    - 1989년 일본 NTT에서 발표
    - **블록 크기** : 128비트
    - **키 길이** : 128비트
- `SNEFRU` :
    - 1990 / R.C.Merkle이 발표
    - 32비트 프로세서에서 구현 용이 목적
    - **블록 크기** : 512비트
    - **키 길이** : 128, 256
