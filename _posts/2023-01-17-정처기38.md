# **Section 38.** 정렬 ( Sort )
---

## 삽입 정렬 ( Insertion Sort )

"이미 순서화된 파일에... n번째 키를 앞의 n-1개의 키와 비교"

가장 간단한 정렬 방식으로 이미 순서화된 파일에 하나의 새 레코드를 순서에 맞게 삽입시켜 정렬

- 시간복잡도는 O(n^2)


## 쉘 정렬 ( Shell Sort )

"매개변수"

임의의 레코드 키와 h값 만큼 떨어진 곳의 레코드 키를 비교하여 순서화되어 있지 않으면 서로 교환하는 것을 반복하는 정렬

- 삽입 정렬을 확장한 개념
- 시간 복잡도 최악 : O(n^2), 평균 : O(n^1.5)


## 선택 정렬 ( Selection Sort )

"n개의 레코드 중에서 최소값을 찾아서"

n개의 레코드 중 최소값을 찾아 첫 번째 레코드 위치에 놓고 나머지 n-1개 중에서 다시 최소값을 찾아 두 번째 레코드 위치에 놓는 방식의 반복
- 시간복잡도는 O(n^2)


## 버블 정렬 ( Bubble Sort )

"인접한 두 개의 레코드"

주어진 파일에서 인접한 두 레코드 키 값을 비교하여 그 크기에 따라 레코드 위치를 교환하는 정렬

- 계속 정렬 여불ㄹ 플래그 비트로 결정
- 시간 복잡도 O(n^2)


## 퀵 정렬 ( Quick Sort )

"하나의 파일을 부분적으로 나누어"

레코드의 많은 자료 이동을 없애고 하나의 파일을 부분적으로 나누어 가면서 정렬하는 방법

- 키를 기준으로 작은 값을 왼쪽, 큰 값을 오른쪽 서브파일로 분해하는 방식
- 정렬 방식 중 가장 빠른 방식
- 위치 관계 없이 임의의 키를 분할 요소로 사용 가능
- 스택이 필요함
- `분할(Divide)`과 `정복(Conquer)`을 통해 자료 정렬
> 분할(Divide) : 기준값인 피봇을 중심으로 정렬할 자료들을 2개의 부분 집합으로 나눔
> 정복(Conquer) : 부분집합의 원소들 중 피봇보다 작은 원소는 좌, 큰 건 우로 정렬
- 더 이상 나눠질 수 없을 때까지 분할 정복을 반복함.
- 시간 복잡도 평균 : O(nlog2 n), 최악 : O(n^2) 


## 힙 정렬 ( Heap Sort )

"완전 이진 트리 (Complete Binary Tree)"

완전 이진 트리를 이용한 정렬 방식

- 시간 복잡도 O(nlog2 n)


## 2-Way 합병 정렬 ( Merge Sort )

"이미 정렬된 두 개의 파일을 한 개의 파일로"

이미 정렬된 두 개의 파일을 한 개의 파일로 합병하는 정렬 방식

- 두 개의 키들을 한 쌍으로 하여 각 쌍에 대하여 순서를 정함
- 순서대로 정렬된 각 쌍의 키들을 합병, 하나의 정렬된 서브리스트로 만듬 > 반복
- 시간 복잡도 O(nlog2 n)


## 기수 정렬 ( Radix Sort ) = Bucket Sort

"버킷"

Queue를 이용하여 자릿수 별로 정렬하는 방식

- 같은 수 같은 문자끼리 그 순서에 맞는 버킷에 분배 했다가 버킷 순서대로 레코드 꺼내서 정렬
- 시간 복잡도 O(dn)
