# **Section 180.** 경로 제어 / 트래픽 제어
---

## 경로 제어 ( Routing )

송, 수신 측 간의 전송 경로 중에서 `최적 패킷 교환 경로`를 결정하는 기능
> `최적 패킷 교환 경로` : 한 경로에 데이터 양이 집중되는거 피하면서 최저 비용으로 최단 시간에 송신할 수 있는 경로
- 경로 제어표를 참조하고 라우터에 의해 수행됨
- **경로 제어 요소** : 성능 기준, 경로의 결정 시간과 장소, 정보 발생지, 경로 정보의 갱신 시간



## 경로 제어 프로토콜 종류 ( Routing Protocol )

효율적인 경로 제어를 위해 네트워크 정보를 생성, 교환, 제어하는 프로토콜

### IGP ( Interior Gateway Protocol )

자율 시스템 내의 라우팅에 사용되는 프로토콜

- `RIP` ( Routing Information Protocol )
    - 가장 널리 사용되는 벡터 라우팅 프로토콜, 최단 경로 탐색에 Bellman-Ford 알고리즘 사용
    - 소규모 동종의 네트워크 내에서 효율적 방법
    - 최대 `홉` 수를 15로 제한 > 대규모 네트워크에선 RIP 사용 못함
    > `홉` : 최종 목적지까지의 과정에서 경유하는 네트워크 수
    - 라우팅 정보 30초 마다 모든 라우터에 알리며 180초 이내 새 정보 수신 안되면 이상 상태로 간주함.

- `OSPF` ( Open Shortest Path First protocol )
    - RIP 단점 해결 > 대규모 네트워크에 사용
    - **라우팅 정보** : 노드 간의 정보, 링크 상태 정보 / 실시간 반영 > 최단 경로 라우팅 지원
    - 다익스트라(Dijkstra) 알고리즘 사용
    - 라우팅 정보 변화가 생길 때만 네트워크의 라우터에 알림
    - 자율 시스템에서 동작하면서 내부 라우팅 프로토콜의 그룹에 도달


### EGP ( Exterior Gateway Protocol )

자율 시스템 간의 라우팅 즉, 게이트웨이 간의 라우팅에 사용되는 프로토콜


### BGP ( Border Gateway Protocol )

- 자율 시스템 간의 라우팅 프로토콜, EGP 단점 보완
- 라우터들이 연결될 때는 전체 경로 제어표를 교환 후 이후에는 변화된 정보만 교환

## 트래픽 제어

네트워크의 보호, 성능 유지, 네트워크 자원의 효율적인 이용을 위해 패킷의 흐름 또는 양을 조절하는 기능


## 트래픽 제어 기법

### 흐릅 제어 ( Flow Control )

네트워크 내의 원활한 흐름을 위해 송 수신 측 사이에 전송되는 **패킷의 양**이나 **속도 규제** 기능

- 송수신측의 처리 속도 또는 버퍼 크기 차이에 의해 생길 수 있는 오버플로 방지 기능

- `정지-대기` ( Stop-and-Wait ) :
    - 수신측 ACK를 받고 다음 패킷을 전송
    - 한 번에 하나의 패킷만 전송 가능

- `슬라이딩 윈도우` ( Sliding Window ) :
    - 확인 신호를 이용해 송신 데이터의 양 조절
    - 수신 측의 확인 신호 없어도 미리 정해진 패킷을 연속적으로 전송 > 한 번에 여러개 전송 가능 > 효율 좋음
    - 최대치를 미리 약속 받음 > 윈도우 사이즈라 함
    - 윈도우 사이즈는 상황에 따라 변함 긍정 수신 응답(ACK)이 전달되면 증가 부정 수신 응답(NCK)이 전달되면 감소


### 폭주 (혼잡) 제어 ( Congestion Control )

네트워크 내의 패킷 수를 조절하여 네트워크 오버플로를 방지

**흐릅 제어와 차이점 (범위)** : 
흐름 제어 > 송수신 사이
폭주 제어 > 네트워크 

- `느린 시작` ( Slow Start )
    - 윈도우 크기를 1,2,4,8~ 2배씩 지수적 증가하여 초기엔 느리지만 갈 수록 빨라짐
    - 데이터 크기가 임계 값에 도달하면 혼잡 회피 단계로 넘어감
- `혼잡 회피` ( Congestion Avoidance )
    - 회피를 위해 윈도우의 크기르 1씩 선형적 증가시켜 혼잡을 예방


### 교착 상태 ( Dead Lock ) 방지

기억 공간이 꽉 차면 다음 패킷이 기억 공간에 들어가기 위해 무한적 기다리는 현상을 방지하기 위한 기법

- 패킷이 같은 목적지를 갖지 않도록 할당
- 교착 상태 발생하면 교착 상태에 있는 한 단말 장치 선택하여 패킷 버퍼 폐기
