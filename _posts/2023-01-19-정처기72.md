# **Section 72.** 인터페이스 보안
---


## 인터페이스 보안

인터페이스는 시스템 모듈 간 통신 및 정보 교환을 위한 통로인데 보안 기능이 없다면 시스템 모듈 전체에 악영향을 줍니다.

그래서 보안성 향상을 위해 보안 취약점을 분석하고 적절한 보안 기능을 적용합니다.


### 인터페이스 보안 취약점 분석

- 기능이 수행되는 구간들의 구현 현황을 확인하고 어떤 **보안 취약점**이 있는지 분석
- **구현 현황**은 송 수신 영역의 구현 기술 및 특징을 구체적으로 확인
- 기능 기반으로 송신 데이터 선택, 송신 객체 생성, 인터페이스 송수신, 데이터 처리 결과 전송 등 영역별로 발생할 수 있는 취약점을 시나리오 형태로 작성


### 인터페이스 보안 기능 적용

인터페이스 기능과 보안 취약점을 기반으로 보안 기능을 적용함.

- **네트워크 영역** :
    - 스니핑 등을 이용한 데이터 탈취 및 변조 위협을 방지하기 위해 트래픽에 대한 암호화 설정
    - 방식 : IPSec, SSL, S-HTTP
- **애플리케이션 영역** :
    - 보안 가이드를 참조해서 코드 상의 보안 취약점을 보완하는 방향으로 보안 기능 적용
- **데이터베이스 영역** :
    - DB, 스키마, 엔티티의 접근 권한과 프로시저 트리거 등 데이터베이스 동작 객체의 보안 취약점에 보안 기능 적용
    - 민감한 데이터 암호화나 익명화하여 자체 보안

## 데이터 무결성 검사 도구

- 시스템 파일의 변경 유무를 확인하고 파일이 변경되었을 때 관리자에게 알리는 도구
- 해커나 크래커가 시스템에 침입하면 백도어를 만들어 놓거나 흔적을 감추는데 무결성 도구로 감지 가능
- 해시 함수를 이용해서 현재 상태를 DB에 저장 후 상태가 달라지면 관리자에게 변경 사실 알림
- 종류 : Tripwire, AIDE, Samhain, Claymore, Slipwire, Fcheck
