# 오답노트
---


## 접근 제어 모델

`BLP 기밀성 모델` : 정보의 불법적인 파괴나 변조보다는 불법적인 비밀 유출 방지에 중점 ( 기밀성 강조 )

`Clark-Wilson 무결성 모델` : 비밀 노출 방지보다는 자료의 변조 방지가 더 중요함 ( 금융, 회계관련 데이터, 기업 재무재표 )

`Chinese Wall` : 충돌을 야기시키는 어떠한 정보의 흐름도 없어야 함, 이익의 충돌 금지

## CMM 모델 ( Capability Maturity Model )

`초기 단계` / `반복 단계` / `정의 단계` / `정량적 관리 단계` / `최적화 단계`

## SPICE 모델

수준 0부터 5까지 총 6개

0단계 `불안정` ( 구현 전, 목적 달성 전)
1단계 `수행` ( 목적이 전반적으로 이뤄짐 )
2단계 `관리` ( 작업 산출물 인도 )
3단계 `확립`( 공학 원칙을 지킴 )
4단계 `예측` ( 산출물의 양적 측정이 가능해짐, 일관된 수행 가능 )
5단계 `최적화`( 지속적으로 업무 목적 만족 )

## 정보시스템

`고가용성 솔루션 ( HACMP )` :
  - High Availability Cluster Multi Processing
  - AIX를 기반, IBM
  - 리소스의 중복 또는 공유를 통해 어플 보호 가능
  - 데이터 공유하거나 동시 접근하는 노드들에서 여러 개의 어플을 실행하게 해줌
  - 두 대 이상의 시스템을 하나의 클러스터로 묶어 클러스터 내의 한 시스템에서 장애가 발생할 경우 다른 시스템이 장애가 발생한 시스템의 자원을 인수할 수 있도록 하여 서비스 중단 최소화

`점대점 연결 방식 ( Point-to-Point Mode )` :
  - 네트워크에 있어 물리적으로는 중개 장치를 통과하지 않고 한 지점에서 다른 지점으로 직접 가는 채널
  - 두 스테이션간을 별도의 회선을 사용하여 1대1로 연결
  - 회선 구성이 간단하고 대용량 전송에 유리
  - 별도의 회선과 포트에 따른 높은 설치비용

`스턱스넷 ( Stuxnet )` :
  - 2010년 6월에 발견된 웜 바이러스
  - 윈도우를 통해 감염, 지맨스 산업의 SW 및 장비를 공격

`루팅 ( Rooting )` :
  - 모바일 기기에서 구동되는 안드로이드 운영체제상에서 최상위 권한을 얻음으로 해당 기기의 생산자 또는 판매자가 걸어 놓은 제약 해제

## 신기술

### DB

- `하둡` : 오픈 소스를 기반으로 한 분산 컴퓨팅 플랫폼
- `디지털 아카이빙` : 디지털 정보 자원을 장기적으로 보존하기 위한 작업
- `타조` : 우리나라 개발 중인 하둡 기반 데이터 웨어하우스
- `데이터 다이어트` : 데이터를 삭제하는 것이 아닌 압축하고 중복 배제하고 새 기준으로 나누어 저장
- `메타 데이터` : 일련의 데이터를 정의하고 설명하는 데이터


### 네트워크

- `비콘` : 
  - 블루투스 기반의 근거리 무선 통신 장치
- `NFC` : 
  - 고주파를 이용한 근거리 무선 통신 기술
- `NDN` : 
  - 콘텐츠 자체의 정보와 라우터 기능만으로 데이터 전송을 수행하는 기술
- `NGN` : 
  - 유선망 기반의 차세대 통신망
- `포스퀘어` : 
  - 위치 기반 소셜 네트워크 서비스
- `스크린 서브넷` : 
  - 외부 네트워크와 내부 네트워크 사이에 도는 완충적인 통신망
- `PLCP` : Physical Layer Convergence Procedure 
  - 논리적인 802.11 MAC 부계층과 물리적인 특성을 연결하는 역할
  - 802.11 MAC 부계층이 물리적 특성에 관계없이 동작하도록 함
- `Traffic Distributor` :
  - 네트워크 통신 간에 트래픽을 분배하는 솔루션
- `DPI` : Deep Packet Inspection
  - 네트워크에서 전송되는 패킷의 헤더와 페이로드 내 정보르 분석하는 컨텐츠 내용 분석 기술
  - 네트워크 보안, 관리, 콘텐츠 관리 등이 목적


### 하드웨어
- `엠디스크` : 한 번의 기록만으로도 자료를 영구 보관할 수 있는 광 저장 장치
- `맴리스터` : 메모리와 레지스터의 합성어, 전류의 방향과 양 등 기존의 경험을 모두 기억하는 소자, 차세대 메모리 기술

## 라우팅 프로토콜

RIP :
  - 자율 시스템 내에서 사용
  - 근거리 통신망 및 독립적 네트워크 내에서 라우팅 정보 관리
  - 가장 많이 사용되는 내부 라우터 프로토콜
  - 최대 홉스 15로 제한

OSPF : 
  - 효율을 위해 자율 시스템을 여러 지역으로 나눔
  - 홉수 카운트 대신 Cost 사용

IGRP : 
  - Cisco System의 고유의 프로토콜
  - 홉수를 기준으로 한 정보 전송
  - 라우팅 경로 결정 시 회선의 전송 능력 지연시간 사용률 신뢰도 바탕
  - 독립적 네트워크 내에서만 사용

EIGRP :
  - IGRP의 Metric 구성 값에 256을 곱하여 작동
  - 프로토콜 종속 모듈
  - 신뢰성 전송 프로토콜 ( 순차적 패킷 전달 )
  - 낮은 대역폭 및 빠른 수렴
  - 업데이트 확산 알고리즘

BGP :
  - 라우터에 의해서 전체 경로 교환
  - 루프 방지
  - 179번 포트 이용한 TCP 서비스 사용
  - 오류 제어나 흐름 제어 필요하지 않음

## 자동화 도구

`SREM`( Software Requirements Enginering Methdology ) :
  - TRW사, 우주 국방 시스템 그룹
  - 실시간 처리 소프트웨어 시스템에서 요구사항을 명확히 기술할 목적으로 개발한 것
  - RSL과 REVS를 사용하는 자동화 도구

`PSL/PSA` :
  - 미시간 대
  - PSL과 PSA를 사용하는 자동화 도구

`SADT` ( Structure Analysis and Design Technique ) :
  - SoftTech사에서 개발된 것
  - 구조적 요구 분석을 위해 블록 다이어그램 채택

## 접근 통제 방법

`RBAC` ( Role Based Access Control ) : 
  - **역할** 기반 접근 통제
  - 권한이 있는 사용자들만 접근할 수 있는 보안 방법
  - 권한부여 : 중앙 관리자

`DAC`( Discretionary Access Control ):
  - 임의 접근 통제
  - 사용자 또는 **신분**
  - 권한부여 : 데이터 소유자

`MAC` ( Mandatory Access Control ): 
  - 강제 접근 통제; 고정적임
  - 미리 정해진 정책과 **보안 등급**
  - 권한부여 : 시스템

합성 중심 ( Composition-Based ) :
  - 전자 칩과 같은 소프트웨어 부품
  - 블록을 만들어서 끼워 맞추어 소프트웨어를 완성시키는 방법
  - 블록 구성 방법

  - 추상화 형태로 쓰여진 명세를 구체화하여 프로그램을 만드는 방법
  - 패턴 구성 방법

## 작성 오류

`생략 오류` ( Omission Error ) :
  - 입력 시 한 자리를 빼놓고 기록한 경우 
  - 1234 > 123

`추가 오류`( Addition Error ) :
  - 입력 시 한 자리 추가로 기록한 경우
  - 1234 > 12345

`필사 오류` ( Transcription Error ) :
  - 입력 시 임의의 한 자리를 잘못 기록한 경우
  - 1234 > 1235

`전위 오류` ( Transposition Error ) : 
  - 입력 시 좌우 자리를 바꾸어 기록한 경우
  - 1234 > 1243

`이중 오류` ( Double Trasposition Error ) :
  - 전위 오류가 두 가지 이상 발생한 경우
  - 1234 > 2143

`임의 오류` ( Random Error ) :
  - 위의 오류가 두 가지 이상 결합하여 발생한 경우
  - 1234 > 12367

## COCOMO 모델

**조직형 (Organic Mode)**
- 중 소규모의 소프트웨어
- 일관 자료 처리, 과학 기술 계산, 비지니스 자료 처리 5만(50KDSI)라인 이하 소프트웨어

**반분리형 (Semi-Detached Mode)**
- 조직, 내장형 중간
- 30만(300KDSI)라인 이하 소프트웨어
- 컴파일러, 인터프리터, 유틸리티

**내장형 (Embedded Mode)**
- 초대형
- 30만(300KDSI)라인 이상
- 시스템 개발

모형 종류

기본형(Basic) : 소프트웨어 크기, 개발 유형만을 이용해서 비용 산정

중간형(Intermediate) : 기본형의 공식 토대, 젶ㅁ, 컴퓨터 개발 요원, 프로젝트 특성 이용하여 비용 산정

발전형(Detailed) : 중간형 보완, 개발 공정별 자세하고 정확하게 노력 산출

## PERT ( Program Evaluation and Review Technique, 프로그램 평가 및 검토 기술 )

- 프로젝트에 필요한 전체 작업의 상호 관계를 표시하는 네트워크
- 낙관적인 경우, 가능성이 있는 경우, 비관적인 경우로 나누어서 단계별 종료 시기를 결정하는 방법
- 경험 없는 프로젝트에 사용
- 노드(작업)와 간선(낙관치, 기대치, 비관치)으로 구성
- 결정 경로, 경계 시간, 작업 간의 상호 관련성 알 수 있음

## 프레임워크

- `모듈화` (Modularity) : 캡슐화를 통해 강화하고 설계 및 구현의 변경에 따른 영향 최소화 > **품질 향상**

- `재사용성` (Reusability) : 재사용 가능한 모듈들을 제공 > **개발자 생산성 향상**

- `확장성` (Extensibility) : 다형성을 통한 인터페이스 확장 가능 > **다양한 개발**

- `제어의 역흐름` (Inversion of Control) : 개발자가 관리하고 통제해야 하는 객체들이 제어를 프레임워크에 넘김 > **생산성 향상**

- 사용자 코드에서 호출해서 사용하는 것이 아닌(이건 라이브러리) 이미 정해진 코드를 호출해서 사용 > 자체적인 흐름을 갖고 있음
