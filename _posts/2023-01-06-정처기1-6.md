# **Section 06.** 요구사항 정의
---

## 요구사항이란

어떤 문제를 해결하기 위해 필요한 조건이나 제약 사항을 요구하는 것

### 요구사항 특징

- 개발이나 유지보수 과정에서 필요한 기준과 근거를 제공함.
- 이해관계자들 간의 의사소통을 원할하게 하는데 도움을 줌.
- 요구사항이 제대로 되어야 이후 과정의 목표와 계획을 수립할 수 있음.

### 요구사항의 유형

- **기능 요구사항** : 기능에 관한 요구사항
    - 어떤 기능, 필수 기능, 제공받기 원하는 기능
    - 입출력, 저장, 연산
- **비기능 요구사항** : 품질이나 제약사항 관련 요구사항
    - 장비 구성 : 시스템 장비 구성
    - 성능 : 처리 속도 및 시간, 처리량, 가용성 등
    - 인터페이스 : 하드웨어, 통신 인터페이스, 프로토콜
    - 데이터 : 보안이 필요한 데이터, 데이터 변환 대상 등
    - 테스트 : 테스트하고 점검
    - 보안 : 접근 통제
    - 품질 : 품질 평가 대상에 대한 요구사항(가용성, 정합성, 대응성, 신뢰성, 사용성, 유지 관리성, 이식성, 확장성 보안성 등으로 구분)
    - 프로젝트 관리 : 관리 방법
    - 프로젝트 지원 : 지원 사항이나 방안
    - 제약사항 : 시스템 설계, 구축, 운영과 관련된 기술, 표준, 업무, 법 제도 등 제약조건
- **사용자 요구사항** : 
    - 사용자 관점
    - 친숙한 표현으로 쉽게 작성
- **시스템 요구사항** : 
    - 개발자 관점
    - 전문적 용어로 기술
    - 소프트웨어 요구사항이라고도 함.


## 요구사항 개발 절차

>`도출` > `분석` > `명세` > `확인`

요구사항 개발 절차는 개발 대상에 대한 요구사항을 체계적으로 `도출`하고
이를 `분석`한 후 분석 결과를 명세서에 `정리`하여 확인 및 검증하는 활동


### 요구사항 도출

시스템, 사용자, 개발자들이 서로 의견을 교환하며 이해하는 과정

- 문제 이해 단계
- 이해관계자가 식별된다.
- 효율적인 의사소통이 중요함
- 소프트웨어 개발 생명 주기동안 지속적으로 반복됨.
- 주요 기법 : 청취, 인터뷰, 설문, 브레인스토밍, 워크샵, 프로토타이핑, 유스케이스 등 질문 기술이 중요

### 요구사항 분석

요구사항 중 모호한 부분을 발견하고 걸러내기 위한 과정

- 타당성 조사 및 비용과 일정에 대한 제약 설정
- 상충되는 요구사항을 중재
- 소프트웨어 범위 파악
- 주변 환경과 상호 작용 방법 이해
- 분석 도구 : 자료 흐름도, 자료 사전, 분석 중재 기술이 중요

### 요구사항 명세

요구사항 바탕으로 모델 작성 및 문서화 

- 문서화할 때 기능 요구사항은 빠짐없이 완벽하게 작성 비기능 요구사항은 필요한 것만 명확하게 작성
- 사용자가 이해하기 쉽고 개발자가 효과적으로 설계할 수 있도록 작성
- 설계 과정에서 잘못된 부분 정의서에서 추적
- 소단위 명세서가 사용될 수도
- 관찰 및 모델 작성 기술이 사용됨.

### 요구사항 명세 기법
- **정형 명세 기법**
    - 수학적 원리 기반, 모델 기반
    - 수학적 기호, 정형화된 표기법
    - 정확, 간결 일관성 있어서 완전성 검증 가능
    - 표기법 어려워서 일반 사용자 어려움
    - 종류 : VDM, Z, Petri-net, CSP 등
- **비정형 명세 기법**
    - 상태, 기능, 객체 중심
    - 명사, 동사 등 자연어 기반으로 작성
    - 일관성 떨어짐,
    - 이해하기는 쉬워 의사소통에 용이
    - 종류 : FSM, Decision Table, ER모델링, State Chart 등

### 요구사항 확인

요구사항 명세서가 정확하게 작성됐는지 검토

- 실제 요구를 반영하는지, 상충되는 요구사항 없는지 점검
- 개발 완료 후 문제 발견하면 재작업 비용이 들어 점검 중요
- 명세서 내용이 이해하기 쉬운지, 일관성은 있는지, 회사 기준에 맞는지, 누락 기능 없는지 검증
- 이해관계자들이 검토
- 단점 :모든 문제 확인할 수는 없음
- 요구사항 관리 도구를 이용해서 정의 문서들에 대해 형상 관리를 수행
> `형상 관리` : 소프트웨어 개발 단계에서 만들어지는 프로그램, 문서 데이터 등을 통칭하여 '형상'이라고 함. 이를 관리하는 것
