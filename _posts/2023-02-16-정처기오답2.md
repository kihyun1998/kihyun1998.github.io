## FTR ( 정형 기술 검토 )

- 오류 검출에 초점을 두고 해결책 나중에 미룸 ( 제품 검토의 집중성 )
- 검토를 위한 자료를 사전에 배포하여 검토 ( 사전 준비성 )
- 의견을 제한하되 충분히 받아들임 ( 의제의 제한성 )
- 안건을 세우며 고수 ( 안건 고수성 )
- 논쟁과 반박을 제한 ( 논쟁 반박의 제한성 )
- 문제 영역을 공개 ( 문제 공개성 )
- 참가자 수 제한 ( 참가 인원의 제한성 )
- 발견된 오류는 문서화 ( 문서성 )

## 테스트

`테스트 케이스` : 사용자 요구사항을 준수했는지 확인하기 위해 설계된 입력 값, 설계 조건, 기대 결과 등에 대한 명세서

`테스트 시나리오` : 테스트 케이스를 적용하는 순서에 따라 여러 개의 테스트 케이스들을 묶은 집합

`테스트 오라클` : 테스트 결과가 올바른지 판단하기 위해 사전 정의된 참 값을 대입하여 비교하는 기법
    - **종류** : 참 오라클, 샘플링 오라클, 추정 오라클, 일관성 오라클

## 개발 단계에 따른 애플리케이션 테스트

- `단위 테스트`
  - 최소 단위에 집중해서 테스트

- `통합 테스트`
  - 단위 테스트 후 모듈 결합하는 과정 테스트
  - **상향식 통합 테스트** : 
    - 프로그램의 하위 모듈에서 상위 모듈 방향으로 통합
    - 하나의 주요 제어 모듈과 관련된 종속 모듈의 그룹인 클러스터가 필요함
  - **하향식 통합 테스트**
    - 상위 모듈에서 하위 모듈 방향으로 통합
    - 깊이 우선 통합법, 넓이 우선 통합법 사용
    - 초기부터 사용자에게 시스템 구조를 보여줌

- `시스템 테스트`
  - 소프트웨어가 컴퓨터 시스템에 완벽 수행되는지 테스트 (기능적, 비기능적 요구사항)

- `인수 테스트`
  - 사용자의 요구사항을 충족하는지 테스트
  - 알파 테스트, 베타 테스트
  - 사용자 인수 테스트, 운영상의 인수 테스트, 계약 인수 테스트, 규정 인수 테스트

- `텍스트 드라이버`
  - 상향식 통합 테스트
  - 하위 모듈 호출

- `테스트 스텁`
  - 하향식 통합 테스트
  - 시험용 모듈

## NS Chart ( Nassi-Schneiderman )

- 3가지 기본구조만으로 논리를 표현(표준화 기능)
- 논리 기술에 중점을 둔 도형을 이용한 표현 방법
- 순차, 반복, 선택, 다중 선택 구조 등 표현
- GOTO나 화살표 사용하지 않음, 선택과 반복 구조를 시각화
- 조건이 복합되어 있는 곳의 처리를 시작적으로 명확히 식별
- 기본구조의 입구와 출구는 각 하나씩
- 전체적인 알고리즘을 일목요연하게 볼 수 있음

## 정렬

`퀵 정렬` : 레코드의 많은 자료 이동을 없애고 하나의 파일을 부분적으로 나눠가며 정렬하는 방식

`삽입 정렬` : 모든 요소를 정렬된 부분과 비교하여 정렬하는 방식

`쉘 정렬` : 삽입 정렬의 확장 개념, 입력 파일을 매게변수값으로 서브파일을 구성하고 서브파일을 삽입정렬 방식으로 순서 배열하는 과정 반복

`선택 정렬` : 최솟값 찾아서 최솟값 위치에 넣는 정렬

`버블 정렬` : 인접한 값 비교하여 교환하는 정렬

`힙 정렬` : 전 이진 트리를 이용한 정렬 방식

`2-Way 합병 정렬` : 이미 정렬된 두 파일을 한 개의 파일로 합병하는 정렬 방식

`기수 정렬` : 키 값을 분석해서 같은 값끼리 순서에 맞게 버킷에 분배하고 버킷의 순서대로 레코드 꺼내 정렬

## 소프트웨어 버전 관리 도구

`RCS ( Revision Control System )`
- CVS와 차이점은 소스 파일의 수정을 한 사람만으로 제한
- 다수의 사용자가 동시에 파일 수정을 할 수 없도록 파일 잠금 방식으로 버전 관리
- 다른 방향으로 진행된 개발 결과를 합치거나 변경 내용을 추적

`RPC ( Remote Procedure Call )`
- 별도의 원격 제어를 위한 코딩 없이 다른 주소 공간에서 리모트의 함수나 프로시저를 실행할 수 있게 해주는 프로세스간 통신


---


## EAI ( Enterprise Application Integration )

기업 내 각종 애플리케이션 및 플랫폼 간의 정보 전달 연계 통합 등 상호연동이 가능하게 해주는 솔루션

- `Point-to-Point` : 가장 기본적인 애플리케이션 통합 방식 1:1로 연결

- `Hub & Spoke` : 단일 접점인 허브 시스템을 통해 데이터 전송하는 중앙 집중형 방식

- `Message Bus` : 애플리케이션 사이에 미들웨어를 두어 처리하는 방식

- `Hybrid` : Hub & Spoke와 Message Bus 혼합 방식

## 소프트웨어 재공학

`Analysis` : 기존 소프트웨어를 **분석**하여 재공학 대상을 **선정**

`Migration` : 기존 소프트웨어를 ***다른 운영체제***나 **하드웨어 환경**에서 사용할 수 있도록 **변환**하는 작업

`Restructuring` : 기존 소프트웨어를 향상시키기 위해 **코드를 재구성**하는 작업

`Reverse Engineering` : 기존 소프트웨어를 분석하여 **소스코드를 얻어**내는 작업

## IPSec (Internet Protocoll Security)

- 양방향 암호 방식
- 네트워크 계층에서 IP 패킷 단위의 데이터 변조 방지 및 은닉 기능을 제공하는 프로토콜
- 발신지 인증, 무결성, 기밀성 보장 ESP 지원
- 발신지 호스트를 인증하고 무결성을 보장하는 인증 헤더를 지원
- 두 게이트웨이 사이에서 IP 패킷 전체를 보호하는 터널모드(Tunnel Mode)와 두 호스트 사이에서 IP 헤더를 제외한 나머지 부분을 보호하는 전송 모드(Transport Mode)로 구분된다.

## 해싱 함수

`제산법 (Division)` : 레코드 키를 해시표의 크기보다 큰 수 중에서 가장 작은 소수로 나눈 나머지를 홈 주소로 삼는 방식 즉 h(K) = K mod Q임
`제곱법 (Mid-Square)` : 레코드 키 값을 제곱한 후 그 중간 부분의 값을 홈 주소로 삼는 방식
`폴딩법 (Folding)` : 레코드 키 값을 여러 부분으로 나눈 후 각 부분의 값을 더하거나 XOR한 값을 홈주소로 삼는 방식
`기수 변환법 (Radix)` : 키 숫자의 진수를 다른 진수로 변환시켜 주소 크기를 초과한 높은 자릿수는 절단하고 이를 다시 주소 범위에 맞게 조정하는 방법
`대수적 코딩법 (Algebraic Coding)` : 키 값을 이루고 있는 각 자리의 비트 수를 한 다항식의 계수로 간주하고 이 다항식을 해시표의 크기에 의해 정의된 다항식으로 나누어 얻은 나머지 다항식의 계수를 홈 주소로 삼는 방식
`계수 분석법 (숫자 분석법)` : 키 값을 이루는 숫자의 분포를 분석하여 비교적 고른 자리를 필요한 만큼 택해서 홈 주소로 삼는 방식
`무작위법 (Random)` : 난수를 발생시켜 나온 값을 홈 주소로 삼는 방식

## 소스 코드 품질 분석

소스 코드의 코딩 스타일, 코드에 설정된 코딩 표준, 코드의 복잡도, 코드에 존재하는 메모리 누수 현상, 스레드 결함 등을 발견하기 위해 사용하는 분석 도구

### 정적 분석 도구

- 작성한 소스 코드를 **실행하지 않고** `코딩 표준`이나 `코딩 스타일`, `결함` 등을 확인
- 개발 완료된 시점에 품질 검증 차원에서 사용
- 자료 흐름, 논리 흐름을 분석하여 비정상적인 패턴을 찾을 수 있음
- 종류 : `pmd`, `cppcheck`, `SonarQube`, `checkstyle`, `ccm`, `cobertura`


### 동적 분석 도구

- 작성한 소스 코드를 **실행하여** 코드에 존재하는 `메모리 누수`, `스레드 결함` 등을 분석
- 종류 : `Avalache`, `Valgrind` 등

## 이진 탐색

1. 배열의 중간 값 찾음
2. 중간 값과 검색 값 비교
  1. 중간 값이 검색 값과 같으면 끝
  2. 중간 값보다 검색 값이 크면 오른쪽 구간 탐색 (low값 교체)
  3. 중간 값보다 검색 값이 작다면 왼쪽 구간 탐색(high값 교체)
3. 값을 찾거나 간격이 비었을 때까지 반복

low, high, middle값을 사용해서 찾음

## IPC (Inter-Process Communication)

모듈 간 통신 방식을 구현하기 위해 사용되는 프로그래밍 인터페이스 집합

Shared Memory : 다수 프로세스 공유 가능한 메모리 구성, 프로세스 간 통신

socket : 소켓 이용해서 프로세스 간 통신

semaphores : 공유 자원에 대한 접근 제어를 통해 프로세스 간 통신

pipes&named pipes : pipe라고 불리는 선입선출 형태로 구성된 메모리 여러 프로세스 공유 통신

message Queueing : 메시지 발생하면 전달 통신
