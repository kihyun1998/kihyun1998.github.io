# **Section 154.** 가상기억장치 구현 기법 / 페이지 교체 알고리즘
---

## 가상기억장치란?

보조기억장치의 일부를 주기억장치처럼 사용하는 것

- 프로그램을 여러 개의 작은 블록 단위로 나눠서 가상기억장치에 보관하고 실행 시 주기억장치에 불연속적으로 할당하여 처리
- 주기억장치보다 큰 프로그램을 사용하려고 씀
- 주기억장치의 이용률과 다중 프로그래밍의 효율을 높일 수 있음
- 가상저장장치의 프로그램 실행하려면 **주소 변환 작업**이 필요하다.
- 블록단위 저장 > 연속 할당 방식에서 발생할 수 있는 단편화 해결 가능


---


## 가상기억장치 기법들

### 페이징(Paging) 기법

프로그램 영역과 주기억장치의 영역을 동일한 크기로 나눈 후 적재시키는 기법

- 페이지(Page) : 프로그램을 일정한 크기로 나눈 단위
- 페이지 프레임(Page Frame) : 페이지 크기로 일정하게 나눠진 단위
- 외부 단편화는 발생 안하지만 **내부 단편화** 발생
> `내부 단편화` : 공간 남는거
- 주소 변환을 위해 페이지 맵 테이블(Page Map Table)이 필요함 ( 비용 증가, 처리 속도 감소)


### 세그멘트(Segmentatin) 기법

다양한 크기의 논리적 단위로 나누어 주기억장치에 적재시켜 실행

- 세그멘트(Segment) : 프로그램, 배열 함수를 논리적 크기로 나눈 단위
- 사용자 관점으로 보존하는 기억장치 관리 기법
- **궁긍적 이유** : 기억공간을 절약하기 위해
- 주소변환을 위해 세그멘트 맵 테이블이 필요함
- 세그멘트가 주기억장치에 적재될 때 다른 세그멘트에게 할당된 영역을 침범할 수 없음 > 기억장치 보호키(Storage Protection Key) 필요
- 내부 단편화는 발생안하지만 외부 단편화 발생


### 세그멘테이션 기법의 일반적인 주소 변환

- **가상주소 형식** : `세그멘트 번호(s)` + `변위값(d)`
> `변위값` : 세그멘트 내에 실제 내용이 위치하고 있는 곳까지의 거리

- **실기억주소 형식** : `(세그멘트 시작주소 + 변위값)`

- **세그먼트 맵 테이블** : `세그멘트 번호(s)` + `세그멘트 크기(L)` + `기준번지(b)`

**주소변환 순서**

1. 세그먼트 번호로 세그먼트 맵 테이블에서 해당하는 세그먼트의 기준번지와 세그먼트 크기를 구함( 세그먼트 번호는 세그먼트 맵 테이블에 대한 색인으로 사용됨 )
2. 가상주소의 변위값과 세그먼트의 크기 비교
3. 변위값이 작거나 같으면 기준번지와 변위값을 더하여 실기억주소로 만들어 주기억장치에 엑세스
4. 변위값이 크면 다른 영역을 침범하게 되므로 실행 권한을 운영체제에게 넘기고 트랩을 발생
   ( 변위값이 크다는 것은 현재 찾는 세그먼트의 위치가 해당 세그먼트의 크기를 초과했다는 의미 )


### 페이지 교체 알고리즘

페이지 부재가 발생하면 가상기억장치의 필요한 페이지를 주기억장치에 적재해야 하는데 어떤 프레임을 선택하여 교체할지 선택하는 기법


- **OPT(OPTimal replacement, 최적 교체)**
1. **앞으로 가장 오랫동안 사용하지 않을** 페이지를 교체하는 기법
2. 벨레이디가 제안한 것으로 페이지 부재 횟수가 가장 적게 발생하는 가장 효율적인 알고리즘

- **FIFO(First in First Out)**
1. FIFO는 각 페이지가 주기억장치에 적재될 때마다 그때의 시간을 기억시켜 **가장 먼저 들어와서 가장 오래 있었던 페이지**를 교체하는 기법
2. 이해하기 쉽고 프로그래밍 및 설계가 간단

- **LRU(Least Recently Used)**
1. **최근에 가장 오랫동안 사용하지 않은** 페이지를 교체하는 기법
2. 페이지마다 카운터나 스택을 두어 가장 오랫동안 사용하지 않은 오래 전에 사용된 페이지를 교체함

- **LFU(Least Frequently Used)**
1. **사용 빈도가 가장 적은 페이지**를 교체하는 기법
2. 활발하게 사용되는 페이지는 사용횟수가 많아 교체되지 않고 사용됨

- **NUR(Not Used Recently)**
1. LRU와 비슷한 알고리즘, **최근에 사용하지 않은 페이지**를 교체하는 기법
2. 최근에 사용되지 않은 페이지는 향후에도 사용되지 않을 가능성이 높다는 것을 전제함. > 시간적인 오버헤드를 줄일 수 있음
3. 사용 여부를 확인하기 위해 `참조 비트`와 `변형 비트`가 사용됨
    **(참조비트, 변형비트, 순서)**
    1. (0,0,1)
    2. (0,1,2)
    3. (1,0,3)
    4. (1,1,4)

- **SCR(Second Chance Replacment, 2차 기회 교체)**
1. 가장 오랫동안 주기억장치에 있던 페이지 중 자주 사용되는 페이지의 교체를 방지하기 위한 것
2. FIFO의 단점을 보완
