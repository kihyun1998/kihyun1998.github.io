# **Section 161.** 인터넷
---

## 인터넷이란?

TCP/IP 프로토콜을 기반으로 하여 네트워크들이 연결된 광범위한 컴퓨터 통신망

- ARPANET에서 시작됨
- 유닉스 운영체제를 기반으로 함
- 고유한 IP주소를 갖음
- 네트워크 연결을 위해서는 브리지, 라우터, 게이트웨이가 사용됨
- 백본 : 네트워크 연결하여 중추적 역할을 하는 네트워크, 인터넷의 주가 되는 기간망


## IP 주소

숫자로 8비트씩 4부분, 32비트로 구성됨

|class|설명|
|--|------|
|A class| - 국가나 대형 통신망에 사용(0~127로 시작)<br> - 2^24 = 16777216개의 호스트 사용 가능|
|B class| - 중대형 통신망에 사용(128~191로 시작)<br> - 2^16 = 65536개의 호스트 사용 가능|
|C class| - 소규모 통신망에 사용(192~223으로 시작)<br> - 2^8 = 256개의 호스트 사용 가능|
|D class| - 멀티캐스트용으로 사용(224~239로 시작)|
|E class| - 실험적 주소이며 공용되지 않음

## 서브넷팅(Subnetting)

할당된 네트워크 주소를 다시 여러 개의 작은 네트워크로 나누어 사용하는 것

- 4 바이트의 IP 주소 중 네트워크 주소와 호스트 주소를 구분하기 위한 비트를 **서브넷 마스크**라고 함

## IPv6 (Internet Protocol version 6)

IPv4의 주소 문제를 해결하기 위해 개발됨

- 주소 길이 : 128비트
- 자료 전송 속도가 빠름
- IPv4와 호환성도 뛰어남
- 인증성, 기밀성, 데이터 무결성 지원 > 보안 문제 해결
- 주소의 확장성, 융통성, 연동성이 뛰어나며 실시간 흐름 제어로 향상된 멀티미디어 기능을 지원함.
> `Traffic Class` : IPv6 패킷의 클래스나 우선순위를 나타내는 필드
> `Flow Lable` : 네트워크 상에서 패킷들의 흐름에 대한 특성을 나타내는 필드
- 패킷 크기를 확장할 수 있어서 패킷 크기에 제한이 없다.
- 기본 헤더 뒤에 확장 헤더를 더하여 더욱 다양한 정보의 저장이 가능해져 네트워크 기능 확장이 용이함
- 자동으로 네트워크 환경 구성이 가능하다.


### IPv6 구성

- 16비트씩 8부분, 총 128비트로 구성되어 있음
- 각 부분을 16진수로 표현하고 콜론으로 구분
- 3가지 주소 체계로 나눠짐

> `유니캐스트(Unicast)` : 단일 송신자와 단일 수신자 간의 통신(1 대 1 통신에 사용)
> `멀티캐스트(Multicast)` : 단일 송신자와 다중 수신자 간의 통신 (1 대 다 통신에 사용)
> `애니캐스트(Anycast)` : 단일 송신자와 가장 가까이 단일 수신자 간의 통신 (1 대 1 통신에 사용)

## 도메인 네임(Domain name)

IP 주소를 사람이 이해하기 쉬운 문자 형태로 표현한 것

- 호스트 컴퓨터 이름, 소속 기관 이름, 소속 기관의 종류, 소속 국가명 순으로 구성되며 왼쪽에서 오른쪽으로 갈 수록 상위 도메인을 의미함
- 문자로 된 도메인 네임을 컴퓨터가 이해할 수 있는 IP 주소로 변환하는 역할을 하는 시스템을 DNS라고 함 이 서버를 DNS 서버라고 함
