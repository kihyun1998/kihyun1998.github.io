## 품질 요구사항

소프트웨어의 품질은 기능, 성능, 만족도 등 소프트웨어에 대한 요구사항이 얼마나 충족되는지를 나타내는 특성의 집합

## ISO/IEC 9126

- 소프트웨어 품질 특성과 평가를 위한 국제 표준이였음
- 품질에 대한 요구사항 기술
- 개발 중이거나 완료된 소프트웨어의 품질 평가
- 2011년에 보안성 강화로 ISO/IEC 25010으로 업데이트

### ISO/IEC 9126 특성

- 효율성

## ISO/IEC 9126 특성 자세히

### 기능성 ( Functionality )

소프트웨어가 사용자의 요구사항을 정확하게 만족하는 기능을 제공하는지

- 적절성 / 적합성 ( Suitability ) : 작업, 목적 달성을 위해 `적절한 기능`을 제공할 수 있는 능력
- 정밀성 / 정확성 ( Accuracy ) : 사용자가 요구하는 결과를 `정확하게 산출`하는 능력
- 상호 운용성 ( Interoperability ): 다른 시스템과 `어울릴` 수 있는 능력
- 보안성 ( Security ) : 권한에 따른 `접근 제어`할 수 있는 능력
- 준수성 ( Compliance ) : 기능과 관련된 `표준, 관례, 규정을 준수`할 수 있는 능력

### 신뢰성 ( Reliability )

요구된 기능을 정확하고 일관되게 오류 없이 수행할 수 있는 정도

- 성숙성 ( Maturity ) : 결함으로 `고장을 피해`갈 수 있는 능력
- 고장 허용성 ( Fault Tolerance ) : 결함 또는 인터페이스 결여에도 규정된 `성능을 유지`할 수 있는 능력
- 회복성 ( Recoverability ) : 고장 시 `다시 회복`하고 영향 받은 `데이터를 복구` 할 수 있는 능력

### 사용성 ( UseAbility )

쉽게 배우고 사용할 수 있는 정도

- 이해성 ( Understandability ) : 사용자가 이해할 수 있는 능력
- 학습성 ( Learnability) : 학습할 수 있도록 하는 능력
- 운용성 ( Operability ) : 운용하고 제어할 수 있는 능력
- 친밀성 ( Attractiveness ) : 다시 사용하고 싶어 하도록 하는 능력

### 효율성 ( Efficiency )

할당된 시간 동안 한정된 자원으로 얼마나 빨리 처리 하는 지

- 시간 효율성 ( Time Behavior ) : 적절한 반응 시간 및 처리 시간, 처리율을 제공할 수 있는 능력
- 자원 효율성 ( Resource Behavior ) : 적절한 자원의 양과 종류 제공할 수 있는 능력

### 유지 보수성 ( Maintainability )

새 요구사항이 발생했을 때 개선하거나 확장할 수 있는 정도

- 분석성 ( Analyzability ) : 결함, 고장원인, 수정될 부분을 식별 가능하게 하는 능력
- 변경성 ( Changeability ) : 환경 변화로인한 수정을 쉽게 구현할 수 있는 능력
- 안정성 ( Stability ) : 변경으로 예상치 못한 결과를 최소화하는 능력
- 시험성 ( Testability ) : 변경이 검증될 수 있는 능력

### 이식성 ( Portability )

다른 환경에서도 쉽게 적용될 수 있는 정도

- 적용성 ( Adaptability ) : 다른 환경으로 변경될 수 있는 능력
- 설치성 ( Instalability ) : 설치할 수 있는 능력
- 대체성 ( Replaceability ) : 동일한 상황에서 다른 소프트웨어를 대신 사용될 수 있는 능력
- 공존성 ( Co-existence ) : 자원 공유 환경에서 다른 소프트웨어와 공존할 수 있는 능력
