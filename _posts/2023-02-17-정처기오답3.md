# 오답노트
---

## 반정규화 (Denormalization)

- `테이블 통합`
  - 검색 간편 but 처리량 증가
  - 입력, 수정, 삭제 복잡해짐

- `테이블 분할`
  - **수평 분할** : 레코드별 사용 빈도 차이가 크면 진행
  - **수직 분할** : 속성 많으면 진행
  - 기본키 유일성 어려워짐
  - 수행 속도 느려질 수도

- `중복 테이블 추가`
  - 정규화로 수행속도 느려지거나 많은, 특정 범위 데이터 자주 처리해야 하거나, 처리 범위 줄이지 않고 속도 개선이 안되면 진행
  - **집계 테이블 추가** : 트리거 설정, 트리거 오버헤드 조심
  - **진행 테이블 추가** : 이력 관리 목적
  - **특정 부분 테이블 추가** : 테이블 특정 부분만 사용

- `중복 속성 추가`
  - 조인 자주 발생, 접근 경로 복잡, 액세스 조건 자주 사용, 기본키 적절하지 않은 경우
  - 테이블, 속성 중복 고려
  - 무결성 유의, 공간 낭비 고려

## 스키마

`외부 스키마` : 개인 또는 응용 개발자 입장에서 보는 데이터베이스
`내부 스키마` : 물리적 저장 장치의 입장에서 본 데이터베이스 구조, 저장될 레코드의 형식 정의 및 저장될 항목의 표현 방법 등을 나타냄
`개념 스키마` : 모든 응용 프로그램 또는 사용자들이 필요로 하는 조직 전체 데이터베이스로 단 하나만 존재함


---


## 물리적 데이터베이스 설계

- 물리적 데이터베이스 설계는 논리적 설계에 기반하여 설계함
1. 물리적 설계의 목적은 효율적인 방법으로 데이터를 저장하는 것
2. 트랜잭션 처리량과 응답시간, 디스크 용량 등을 고려
3. 저장 레코드의 형식 순서 접근 경로와 같은 정보를 사용
>>2,3은 물리적 요소 고려
4. 트랜잭션의 인터페이스 설계 > 이건 논리적 설계

---


## 물리적 데이터베이스 - 파티션 설계

- **범위 분할** ( Range Partitioning ) : 지정한 열의 값을 기준으로 분할

- **해시 분할** ( Hash Partitioning ) : 해시 함수를 적용한 결과 값에 따라 데이터 분할

- **조합 분할** ( Composite Partitioning ) : 범위 분활 후 해시 함수를 적용하여 다시 분할

- **목록 분할** ( List Partitioning ) : 지정한 열 값에 대한 목록을 만들어 분할

- **라운드 로빈 분할** ( Round Robin Partitioning ) : 레코드를 균일하게 분배 ( 기본키 필요 없음)


---


## 무결성 규정

- **개체 무결성** : 모든 테이블이 기본키로 선택된 칼럼을 가져야 함

- **참조 무결성** : 참조 관계의 두 테이블의 데이터가 항상 일관된 값을 갖도록 유지하는 것

- **도메인 무결성** : 테이블에 존재하는 필드의 무결성을 보장하기 위한 것 (속성 값)


---


## 관계 대수

**일반 집합 연산자**
- 합집합(Unino), 교집합(Intersection), 차집합(Difference), 교차곱(Cartesian Product)

**순수 관계 연산자**
- `Select`, `Project`, `Join`, `Division`

---


## 키

`슈퍼키(Super Key)` : 유일성을 만족하는 속성 또는 속성들의 집합
`후보키(Candidate Key)` : 유일성과 최소성을 만족하는 속성 또는 속성들의 집합
`대체키(Alternate Key)` : 기본키로 선택되지 못한 후보키
`외래키(Foreign Key)`: 다른 릴레이션의 기본키를 참조하는 속성 또는 속성들의 집합

## 데이터웨어하우스의 기본적인 OLAP 연산

`roll-up`, `slicing & dicing`, `drill-up & down`, `pivot`, `drill-through`

>`OLAP` (Online Analytical Processing) : 다차원으로 이루어진 데이터로부터 통계적인 요약 정보를 분석하여 의사 결정에 활용하는 방식

## 병행 제어의 목적

- 여러 사용자들의 데이터베이스 공동 사용을 최대화 ( 공유 최대화 )
- 사용자의 응답 시간 최소화
- 데이터베이스 시스템의 활용도 최대화
- 데이터베이스의 일관성 유지

## 분산 데이터베이스 시스템 ( Distributed Database System )

**주요 구성 요소** : `분산 처리기`, `분산 데이터베이스`, `통신 네트워크`, `분산 트랜잭션`
`분산 데이터베이스`의 구조 > `전역 스키마`, `분할 스키마`, `할당 스키마`, `지역 스키마`

**목표**
- `위치 투명성`, `중복 투명성`, `병행 투명성`, `장애 투명성`

## 시스템 카탈로그

시스템 그 자체에 관련이 있는 다양한 객체에 관한 정보를 포함하는 시스템 데이터베이스
저장된 정보를 `메타 데이터`라고 하고 DBMS가 스스로 생성하고 유지함, `데이터 사전`이라고도 함


---


## 로킹

트랜잭션이 데이터에 접근할 때 로킹 수행
필드, 레코드, 테이블, 파일, db모두 로킹 단위가 될 수 있음


로킹 단위 `크면` : 
  **로크 수** : 작다
  **병행성** : 낮아짐
  **오버헤드** : 낮아짐
  **공유도** : 낮아짐
  간단하고 수월

로킹 단위 `작으면` : 
  **로크 수** : 많다
  **병행성** : 높아짐
  **오버헤드** : 높아짐
  **공유도** : 높아짐
  까다롭고 복잡

## 뷰

- 하나 이상의 테이블로부터 유도되는 가상 테이블로 논리적 독립성이 제공

- 특정 사용자가 볼 필요가 없는 다른 열을 배제하고 뷰를 만든 후 해당 사용자에게 뷰에 대한 권한을 할당하면 테이블 전체에 대한 권한을 부여하지 않아도 됨

- 뷰는 인덱스를 갖지 않아서 삽입, 삭제, 수정에 제한적임


---


## 인덱스

- 데이터베이스 성능에 많은 영향을 주는 DBMS 구성 요소
- 테이블과 클러스터에 연관되어 독립적인 저장 공간을 보유
- 데이터베이스 저장된 자료의 빠른 조회를 위한 순서 데이터
- 테이블 삭제하면 인덱스 같이 삭제됨
- 생성 : CREATE; 삭제 : DROP
