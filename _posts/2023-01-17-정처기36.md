# **Section 36.** 자료 구조
---

## 자료 구조란?

자료 구조는 프로그램에서 사용하기 위한 자료를 기억장치의 공간 내에 저장하는 방법과 그룹 내에 존재하는 자료 간의 관계, 처리 방법 등을 연구 분석하는 것

### 특징

- 자료의 표현과 그것과 관련된 연산
- 일련의 자료들을 조직하고 구조화하는 것
- 어떠한 자료 구조에서도 필요한 모든 연산 처리 가능
- 자료 구조에 따라 프로그램 실행 시간 달라짐

---

## 자료 구조의 분류

**선형 구조**
    - 배열 ( Array )
    - 선형 리스트 ( Linear List )
        - 연속 리스트 ( Contiguous List )
        - 연결 리스트 ( Linked List )
    - 스택 ( Stack )
    - 큐 ( Queue )
    - 데크 ( Deque )

**비선형 구조**
    - 트리 ( Tree )
    - 그래프 ( Graph )


### 배열

동일한 자료형의 데이터들이 같은 크기로 나열되어 순서가 있는 집합

- `정적인 구조` > 기억장소 추가 어려움, 메모리 낭비 발생
- `첨자`를 이용해 데이터에 접근
- `반복적 데이터` 처리 작업에 적합
- 데이터마다 동일한 이름의 변수를 사용해 처리 > `간편`


### 선형 리스트

배열을 이용한 리스트는 `연속 리스트`, 포인터를 이용한 리스트는 `연결 리스트`입니다.

- **연속 리스트**
    - `연속되는 기억장소`에 저장 > 메모리 효율 좋음.
    - 삽입 삭제 시 자료의 이동 필요
- **연결 리스트**
    - 연속적으로 배열시키지 않고 임의의 기억공간에 기억시키되 자료 순서에 따라 노드의 포인터 부분을 이용하여 연결시킨 구조
    - 삽입 삭제 작업이 용이
    - 기억 공간 `연속적이지 않아도` 저장 가능
    - 연결을 위해 포인터 부분 필요하기에 메모리 효율 안좋음.
    - 포인터 찾는 시간이 필요하여 접근 속도 느림
    - 중간에 노드 연결 끊어지면 다음 노드 찾기 어려움


### 스택

리스트의 한 쪽 끝으로만 자료의 삽입 삭제가 가능한 자료 구조

- `후입선출(LIFO)` 방식
- full인 상태에서 데이터 삽입되면 `오버플로우`
- 빈 상태에서 데이터 삭제되면 `언더플로우`
- 응용 분야 : 인터럽트 처리, 복귀 주소 저장 등 ..


### 큐

한 쪽으로는 입력만 다른 한 쪽으로는 출력만 가능한 자료 구조

- `선입선출(FIFO)` 방식
- 시작과 끝을 표시하는 두 개의 포인터 있음
- 운영체제의 작업 스케줄링에 사용


### 그래프

정점과 간선의 두 집합으로 이루어진 자료 구조

- 간선의 방향성에 따라 방향 그래프, 무방향 그래프로 구분됨.
- 통신망, 교통망 등에 응용
- 트리는 사이클이 없는 그래프
- 무방향 그래프 최대 간선 수 : n(n-1)/2
- 방향 그래프 최대 간선 수 : n(n-1)
