# **Section 11.** 사용자 인터페이스
---

## 사용자 인터페이스란?

사용자 인터페이스 (User Interface : UI)는 사용자와 시스템 사이의 원활한 상호작용을 위해서 도와주는 장치 or 소프트웨어 입니다.

### 분야

- 제어
- 기능
- 구성

### 특징

- 사용자 만족도에 가장 많은 영향을 미침
- 소프트웨어 영역에서 변경이 가장 많이 발생함.
- 최소한의 노력으로 원하는 결과를 얻을 수 있게 함.
- 사용자의 편리성과 가독성을 높임. > 작업시간 단축, 업무 이해도 향상
- `사용자 중심으로 설계` : 사용자 중심의 상호작용
- 수행 결과의 오류 줄임



## 사용자 인터페이스 구분

### `CLI` (Command Line Interface)

- 명령과 출력이 텍스트 형태로 이뤄지는 인터페이스

### `GUI` (Graphical User Interface)

- 아이콘이나 메뉴를 마우스로 선택하여 작업을 수행하는 그래픽 환경의 인터페이스
### `NUI` (National User Interface)

- 사용자의 말이나 행동으로 기기를 조작하는 인터페이스
### `VUI` (Voice User Interface)

- 사람 음성으로 기기를 조작하는 인터페이스
### `OUI` (Organic User Interface)

- 모든 사물과 사용자간의 인터페이스, 하드웨어 분야에 사용( VR, AR, IoT, MR )

### 모바일 제스처

- Tap (누르기) : 화면을 터치
- Double Tap (두번 누르기) : 화면을 빠르게 두번 터치
- Drag (누른 채 움직임) : 드래그
- Pan (누른 채 계속 움직임) : 드래그의 연속을 panning이라고 한다
- Press (오래 누르기) : 일명 꾹 누르기
- Fick (빠른 스크롤) : 수평 or 수직으로 빠른 드래그
- Pinch (넓히기/ 좁히기) : 확대 or 축소



## 사용자 인터페이스의 기본 원칙

- 직관성 : 누구나 `쉽게 이해`하고 사용할 수 있어야 한다.
- 유효성 : 사용자의 `목적`을 정확하고 완벽하게 `달성`해야 한다.
- 학습성 : 누구나 `쉽게 배우고` 익힐 수 있어야 한다.
- 유연성 : 사용자의 `요구사항을 최대한 수용`하고 `실수를 최소화`해야 한다.



## 사용자 인터페이스의 설계 지침

- 사용자 중심 : 사용자가 쉽고 편리하게 사용할 수 있는 환경 제공, 실사용자에 대한 이해가 바탕이 되어야 함.
- 사용성 : 사용자가 쉽게 이해할 수 있는지 효율적으로 사용할 수 있는지 확인 > 설계 시 가장 우선적으로 고려
- 일관성 : 버튼, 조작 방법을 일관성 있게 제공 > 사용자가 쉽게 기억하고 습득
- 단순성 : 조작 방법 단순화 > 인지적 부담 감소
- 결과 예측 가능 : 기능만 보고 결과를 예측할 수 있게 설계
- 가시성 : 메인 화면에 주요 기능을 노출 시켜 조작 쉽도록 설계
- 심미성 : 디자인적으로 완벽하게, 가독성 높일 수 있게
- 표준화 : 기능 구조, 디자인을 표준화하여 한 번 학습 이후 쉽게 사용할 수 있도록 설계
- 접근성 : 다양한 계층이 사용할 수 있도록 설계
- 명확성 : 개념적으로 쉽게 인지할 수 있도록 설계
- 오류 발생 해결 : 사용자가 오류를 쉽게 인지할 수 있도록 설계


## 사용자 인터페이스 개발 시스템의 기능

- 사용자 입력 검증
- 에러 처리 and 에러 메시지
- Help and Prompt 제공
