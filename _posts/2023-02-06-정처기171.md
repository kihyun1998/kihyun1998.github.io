# **Section 171.** 수학적 산정 기법
---

## 수학적 산정 기법이란

상향식 산정 기법으로 경험적 추정 모형, 실험적 추정 모형이라고도 하고 개발 비용 산정의 자동화를 목표로 함.

- 경험적으로 유도된 공식들임
- 종류 : COCOMO 모형, Putunam 모형, 기능 점수(FP) 모형이 있고 지정된 공식을 사용해서 비용 산정


---


## COCOMO 모형

보헴이 제안한 LOC에 의한 비용 산정 기법

- 개발할 소프트웨어의 규모(LOC)를 예측하고 소프트웨어 종류에 따라 다르게 측정되는 방정식에 대입하여 비용 산정
- 작은 규모의 프로젝트들을 분석한 결과를 반영한 모델이라 중소 규모 프로젝트 비용 추정에 적합
- 같은 규모의 프로그램이더라도 성격에 따라 비용 다르게 산정됨
- 결과는 완성하는데 필요한 노력(인월)로 나타남


### 조직형 ( Organic Mode )

기관 내부에서 개발된 중소규모의 소프트웨어로 일괄 자료 처리나 과학 기술 계산용, 비지니스 자료 처리용으로 `5만 라인( 50KDSI ) 이하의 소프트웨어`개발 유형

- 사무처리, 업무, 과학용 **응용 소프트웨어** 개발에 적합

**공식**
```
`노력(인월)` = 2.4 * (KDSI)^1.05
> `KDSI` : Kilo Delivered Source Instruction으로 전체 라인수를 1000라인 단위로 묶은 것
`개발 기간` = 2.5 * (노력)^0.38
```

### 반분리형 ( Semi-Detached Mode )

조직형과 내장형의 중간형으로 트랜잭션 처리 시스템이나 운영체제, DBMS 등 `30만 라인( 300KDSI ) 이하의 소프트웨어` 개발 유형

- 컴파일러, 인터프리터 같은 **유틸리티** 개발에 적합

**공식**
```
`노력(인월)` = 3.0 * (KDSI)^1.12
`개발기간` = 2.5 * (노력)^0.35
```

### 내장형 ( Embedded Mode )

초대형 규모의 트랜잭션 처리 시스템이나 운영체제 등의 `30만 라인 (300KDSI) 이상의 소프트웨어` 개발 유형

- 신호기 제어 시스템, 미사일 유도 시스템, 실시간 처리 시스템 등 `시스템 프로그램` 개발에 적합

**공식**
```
`노력(인월)` = 3.6 * (KDSI)^1.20
`개발 기간` = 2.5 * (노력)^0.32
```


## Putnam 모형

소프트웨어 생명 주기의 전 과정 동안에 사용될 노력의 분포를 가정해주는 모형

- 푸트남이 제안한 것으로 생명 주기 예측 모형이라고도 함
- Rayleigh-Norden 곡선의 노력 분포도를 기초로 함.
> `Rayleigh-Norden` : 노든이 소프트웨어 개발에 관한 경험적 자료를 수집하여 이를 근거로 그린 곡선
- 대형 프로젝트의 노력 분포 산정에 이용되는 기법
- 개발 기간이 늘어날 수록 프로젝트 적용 인원의 노력이 감소

**산정 공식**

```
개발 노력 = L^3 / Ck^3 * Td^4

L = 원시 코드 라인 수
Td = 개발 기간
Ck = 환경 상수 ( 빈약 환경 = 2000, 좋은 환경 = 8000, 최적 환경 = 12000)
```


## 기능 점수(FP) 모형

Function Point, 알브레히트가 제안함, 소프트웨어의 기능을 증대시키는 요인별로 가중치 부여하여 합산한 총 기능 점수를 산출해서 영향도를 이용해 비용 산정

**산정 공식**

```
기능 점수 = 총 기능 점수 * [0.65 + (0.1 * 총 영향도)]
```

**기능별 가중치**
|소프트웨어 기능 증대 요인|단순|보통|복잡|
|-----|--|--|--|
|자료 입력 (입력 양식)|3|4|6|
|정보 출력 (출력 보고서)|4|5|7|
|명령어 (사용자 질의수)|3|4|5|
|데이터 파일|7|10|15|
|필요한 외부 루틴과의 인터페이스|5|7|10|


## 자동화 추정 도구

`SLIM` : Rayleigh-Norden 곡선과 Putnam 예측 모델을 기초로 개발된 자동화 측정 도구
`ESTIMACS` : 다양한 프로젝트와 개인별 요소를 수용하도록 FP 모형을 기초로 개발된 자동화 추정 도구
