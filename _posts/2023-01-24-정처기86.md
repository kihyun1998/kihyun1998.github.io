# **Section 86.** 관계형 데이터베이스의 제약 조건 - 무결성
---


## 무결성이란?

데이터베이스에 저장된 데이터 값과 그것이 표현하는 현실세계의 실제 값이 일치하는 정확성

- 무결성 제약 조건은 데이터의 정확성을 보장하기 위해 부정확한 자료가 저장되는 것을 방지하기 위한 제약 조건

---

## 무결성 종류

### 개체 무결성 (Entity Integrity, 실체 무결성)

기본키를 구성하는 어떤 속성도 NULL값이나 중복값을 가질 수 없다.

### 도메인 무결성 (Domain Integrity, 영역 무결성)

주어진 속성 값이 정의된 도메인에 속한 값이어야 한다.

### 참조 무결성 (Referential Integrity)

외래키 값은 NULL이거나 참조 릴레이션의 기본키 값과 동일해야 한다.
즉 참조할 수 없는 외래키 값을 가질 수 없다.

### 사용자 정의 무결성

속성 값들이 사용자가 정의한 제약 조건을 만족해야 한다는 규정

---

## 데이터 무결성 강화

데이터 무결성은 데이터 품질에 영향을 미치기 때문에 데이터 특성에 맞게 무결성을 강화해야 함.

- 프로그램이 완성되고 데이터가 저장된 상태에서 무결성 정의하면 비용이 많이 발생하니까 구축 과정에서 정의한다.


### 애플리케이션

- 데이터 생성, 수정, 삭제 시 무결성 검증하는 코드를 프로그램 내 추가
- 코드를 이용한 검토는 애플리케이션 내에서 처리 (데이터베이스에서 수행 어려움)

**장점** : 사용자 정의 같은 복잡한 무결성 조건 구현 가능
**단점** : 소스코드가 분산되어 관리 힘들고 개별적 시행하기에 적정성 검토 어려움


### 데이터베이스 트리거

- 트리거 이벤트에 무결성 조건을 실행하는 절차형 SQL 추가

**장점** : 통합 관리 가능, 복잡한 요구조건 구현 가능
**단점** : 운영 중 변경 어렵고 사용상 주의가 필요

### 제약 조건

- 데이터베이스에 제약 조건을 설정해서 무결성 유지

**장점** : 통합 관리 가능, 구현이 간단, 변경하기 쉬움, 오류 데이터 방지
**단점** : 복잡한 제약 조건의 구현과 예외적 처리 불가
