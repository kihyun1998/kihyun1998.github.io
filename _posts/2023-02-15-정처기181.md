# **Section 181.** SW 관련 신기술
---

## 인공지능 ( AI; Artificial Intelligence )
- 인간의 두뇌와 같이 컴퓨터 스스로 추론, 학습, 판단 등 인간지능적인 작업을 수행하는 시스템
- **응용 분야** : 신경망, 퍼지, 패턴 인식, 전문가 시스템, 자연어 인식, 이미지 처리, 컴퓨터 시각, 로봇 공학

---

## 뉴럴 링크 ( Neuralink )

- 뇌와 컴퓨터를 결합하는 기술 개발을 위해 설립된 회사
- 작은 전극을 뇌에 이식하여 생각을 업로드하고 다운로드하는 것을 목표로 함

---

## 딥 러닝 ( Deep Learning )

- 인간의 두뇌를 모델로 만들어진 인공 신경망(ANN)을 기반으로 하는 기계 학습 기술
- 사람처럼 스스로 학습할 수 있어서 데이터를 정형화하지 않고 데이터 수집, 분석을 고속 처리 가능

---

## 전문가 시스템 ( Expert System )

- 의료 진단 같은 전문가가 수행하는 업무를 지원하기 위한 응용 프로그램
- 지식 베이스라는 DB와 추론 기구를 활용하여 결정 내림

---

## 가상 현실 ( VR; Virtual Reality )

- 실제와 유사하지만 실제가 아닌 환경이나 상황 구현

---

## 증강 현실 ( AR; Augmented Reality )

- 실제 촬영한 화면에 가상의 정보를 부가하여 보여주는 기술

---

## 혼합 현실 ( MR; Mixed Reality )

- 가상+현실세계 현실의 물리적인 객체와 가상 객체가 상호작용할 수 있는 환경 구현

---

## 그레이웨어 ( Grayware )

- 소프트웨어 제공하는 입장에서 악의적인 목적으로 배포하진 않지만 사용자 입장에서는 악의적이 될 수도 있는 소프트웨어
- ex) 애드웨어, 트랙 웨어, 악성 공유웨어

---

## 매시업 ( Mashup )

- 웹에서 제공하는 정보 및 서비스를 이용하여 새 소프트웨어나 서비스, DB를 만드는 기술

---

## 리치 인터넷 애플리케이션 ( RIA )

- 플래시 애니메이션 기술과 웹 서버 애플리케이션 기술을 통합, HTML 보다 역동적이고 인터랙티브한 웹 페이지 제공

---

## 시맨틱 웹 ( Semantic Web )

- 컴퓨터가 사람을 대신해 정보를 읽고 이해해서 새 정보를 만들 수 있도록 이해하기 쉬운 의미를 가진 차세대 지능형 웹
- 웹 자원을 서술하기 위한 자원 서술 기술, 온톨로지를 통한 지식 서술 기술, 통합적으로 운영하기 위한 에이전트 기술이 있음

---

## 증발품 ( Vaporware )

- 판매 계획, 배포 계획은 발표 했지만 판매되거나 배포되지 않고 있는 소프트웨어

---

## 오픈 그리드 서비스 아키텍처 ( OGSA; Open Grid Service Architechture )

- 공유를 위한 웹 서비스를 그리드 상에서 제공하기 위해 만든 개방형 표준
- 웹 서비스 표준을 적극적으로 따르고 기존의 웹 개발 툴들을 그대로 사용할 수 있다는 장점이 있음

---

## 서비스 지향 아키텍처 ( SOA; Service Oriented Architecture )

- 기업 정보 시스템을 공유와 재사용이 가능한 서비스 단위나 컴포넌트 중심으로 구축하는 정보 기술 아키텍처
- 비지니스에 맞춰 유연하게 사용할 수 있다는 장점이 있음
- **SOA 기반 애플리케이션 구성 계층** : 표현 계층, 업무 프로세스 계층, 서비스 중간 계층, 애플리케이션 계층, 데이터 저장 계층

---

## 복잡 이벤트 처리 ( CEP; Complex Event Processing )

- 실시간 발생 사건들 중 의미 있는 것만 추출할 수 있도록 조건 정의하는 데이터 처리 방법
- 금융, 통신, 전력, 물류, 국방 등에서 대용량 데이터 스트림에 대한 요구 대응을 위해 개발

---

## 디지털 트윈 ( Digital Twin )

- 현실 사물을 소프트웨어로 가상화한 모델
- 실제 자산의 특성에 대한 정확한 정보 획득, 최적화, 사고 최소화, 생산성 증가, 효율성 향상
- 모의 실험 목적으로 사용

---

## 텐서플로 ( TensorFlow )

- 구글 브레인팀이 만든 데이터 흐름 프로그래밍을 위한 오픈소스 소프트웨어 라이브러리
- C++로 제작, 구글 검색, 음성 인식, 번역 등 구글 서비스 전반에서 다양하게 사용됨

---

## 도커 ( Docker )

- 컨테이너 기술을 자동화하여 쉽게 사용할 수 있는 오픈소스 프로젝트
- 소프트웨어 컨테이너 안에 응용 프로그램들을 배치시키는 일을 자동화해주는 역할 수행

---

## 스크래피 ( Scrapy )

- python 기반 웹 크롤링 프레임워크, 코드 재사용성을 높이는 데 도움이 되며 대규모의 크롤링 프로젝트에 적합
